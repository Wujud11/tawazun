Mission ID
----------
M03 — Code Cleanup and Reusable Components


Goal
----
Extract the duplicated KPI stat card grid pattern from three page files into a
single shared component. Do not change any UI behaviour, styling, or data.


Context
-------
The M01 audit confirmed that the following three pages each contain their own
independent implementation of the same KPI card grid pattern:

  src/pages/debts.tsx
    - Local `kpiCards` array (lines ~68–101): { id, label, value, sub, icon, colorClass }
    - Local `KpiCards` function component (lines ~105–129)
    - Renders a 4-column grid of Cards: icon header + large value + small sub-label

  src/pages/netting.tsx
    - Local `kpiCards` array (lines ~48–81): same shape
    - Inline KPI render inside NettingPage (lines ~404–424): same grid structure

  src/pages/companies.tsx
    - Local `summaryCards` array (lines ~56–92): { id, label, value, suffix, icon, colorClass }
      (key is `suffix` instead of `sub` — treat as equivalent)
    - Local `CompanySummaryCards` function (lines ~94–118): same grid structure

The existing shared component src/components/dashboard/kpi-cards.tsx is
dashboard-specific (bound to dashboardKpis data, has change-percentage indicator)
and must NOT be modified or used as the basis for this extraction.

Implementation Tasks
--------------------
1. Read all files listed in "Files to Read" above.

2. Create src/components/ui/stat-cards.tsx.
   Define a StatCard type:
     type StatCard = {
       id: string
       label: string
       value: string
       sub: string
       icon: React.ElementType
       colorClass: string
     }
   Export a StatCardGrid component:
     - Props: { cards: StatCard[] }
     - Renders: <div className="grid gap-4 sm:grid-cols-2 xl:grid-cols-4">
     - Each card: Card > CardHeader (label left, icon right) > CardContent (value, sub)
     - Match the exact className values used in the three page implementations.

3. Update src/pages/debts.tsx.
   - Import StatCardGrid from '@/components/ui/stat-cards'.
   - Replace the local `KpiCards` function component and `kpiCards` array with
     a `kpiCards` const of type StatCard[] (same data, `sub` key unchanged).
   - Replace <KpiCards /> in the JSX with <StatCardGrid cards={kpiCards} />.
   - Remove any imports that are no longer used after this change.

4. Update src/pages/netting.tsx.
   - Import StatCardGrid from '@/components/ui/stat-cards'.
   - The existing `kpiCards` array already uses `sub` — keep it, add StatCard type.
   - Replace the inline KPI card grid in NettingPage JSX with
     <StatCardGrid cards={kpiCards} />.
   - Remove any imports that are no longer used after this change.

5. Update src/pages/companies.tsx.
   - Import StatCardGrid from '@/components/ui/stat-cards'.
   - Rename the `summaryCards` array entries: `suffix` → `sub`.
   - Replace the local `CompanySummaryCards` function with
     <StatCardGrid cards={summaryCards} /> inline at the call site.
   - Remove any imports that are no longer used after this change.

6. Run the build:
   npm run build

7. Run the TypeScript check:
   npx tsc --noEmit

8. If either check fails, fix only the allowed files until both pass.
