# Mission ID
M03 � Code Cleanup and Reusable Components

# Objective
Reduce confirmed code duplication and improve maintainability without changing UI behavior or product functionality.

# Why this mission exists
The M01 audit identified that the KPI card grid pattern � a row of stat cards each containing an icon, label, large value, and sub-label � is implemented three times independently across three page files. Each implementation is structurally identical but written from scratch with no shared abstraction. This violates the single-source principle for UI patterns and makes future changes (layout tweaks, accessibility fixes, theming) require edits in three places instead of one.

The three duplicate implementations are:
1. `src/pages/debts.tsx` � local `kpiCards` array + local `KpiCards` function component (pattern: `{ id, label, value, sub, icon, colorClass }`)
2. `src/pages/netting.tsx` � local `kpiCards` array + inline render inside `NettingPage` (same pattern)
3. `src/pages/companies.tsx` � local `summaryCards` array + local `CompanySummaryCards` function (same pattern, key named `suffix` instead of `sub`)

The existing shared component `src/components/dashboard/kpi-cards.tsx` is NOT reusable for these pages because it is tightly coupled to `dashboardKpis` data and includes a change-percentage indicator (up/down arrow + `changeLabel`) that the three page-level cards do not have. It must not be modified.
